## **سلايد ١٨: ليش تحتاج قتهب وايش هي الـ branches**

**ليش قتهب؟**
لازم ترفع مشروعك على GitHub. هذا يحفظ لك نسخة آمنة، ويخليك تشوف كل تعديل سويته من أول يوم، وتقدر ترجع لأي حالة قديمة بسهولة.

**ايش هي الـ Branches؟**
الـ branch تخيّله مسار جانبي للكود. الـ main branch هو الكود الأساسي. لو بتجرب فكرة جديدة، تسوي branch جديد، تشتغل فيه، ولو طلعت كويسة تدمجها مع الـ main.

**ملاحظة:**
بما إنك تشتغل لحالك، ما تحتاج تتعقّد بموضوع pull requests والـ reviews. هذا للفِرَق.

**فايدتها:**
- تشوف سجل كل التعديلات (history)
- Stage و Unstage للتغييرات قبل ما تحفظها
- ترجع لأي نقطة قديمة لو شي خرب

---

## **سلايد ١٩: تحذير، الأمان في قتهب**

**صفر من ناحية الأمان؟ هنا تقرا بتركيز.**

في قتهب فيه شي اسمه **public repos**، وأي شخص يقدر يشوفها. لو رفعت بيانات حساسة فيها، الناس كلها تشوفها.

**ما ترفع أبدًا:**
- ✗ API Keys (مفاتيح OpenAI, Stripe, AWS, إلخ)
- ✗ كلمات السر و الـ database passwords
- ✗ ملفات .env الكاملة
- ✗ أي توكنز أو مفاتيح خاصة فيك

**الحل:**
- استخدم ملف **.gitignore** يخفي الملفات الحساسة من الرفع
- خلي ملف .env في .gitignore دايمًا
- لو غلط رفعت مفتاح، **غيّره فورًا** من المنصة (مو بس تحذفه من قتهب، التاريخ يحفظه)

**قاعدة:** قبل أول push، شيك على ملفاتك مرتين. خصوصًا .env و الـ config files.
